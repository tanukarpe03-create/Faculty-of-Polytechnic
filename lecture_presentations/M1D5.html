>> Recap 01 / 04 · Variables & types

A box, a label, and a kind of thing

A variable is a labelled box that holds a value. Every value has a type — the four you'll meet constantly:

types.py
name    = "Aisha"     # str  — text
age     = 19          # int  — whole number
height  = 1.62        # float — decimal
enrolled = True       # bool — True / False
Types matter.
The type decides what you can do with a value. You can add two ints; you can't add an int to a str without converting first.
>> Recap 02 / 04 · Input + conversion

input() is always a string — convert it

convert.py
# input() hands you text — every time
age_text = input("How old are you? ")
# age_text is the str "19", not the number 19

# Wrap it to get a real number:
age = int(input("How old are you? "))   # whole number
pay = float(input("Hourly rate? "))   # decimal
The reflex.
Want a number from the user? int(input(...)) or float(input(...)). Forget the wrapper and your maths breaks.
>> Recap 03 / 04 · f-strings

f-strings — drop values straight into text

fstrings.py
name = "Aisha"
city = "Pune"
years = 2

# Put an f in front, then {variables} inside the string:
print(f"{name} is from {city}.")
# → Aisha is from Pune.

# You can do maths inside the braces too:
print(f"In 3 years: {years + 3} years of coding.")
# → In 3 years: 5 years of coding.

>> Common failures

Five Week-1 mistakes to avoid today

  1. Forgetting int() around input() when you need a number.
  2. Forgetting the f before a string — "Hi {name}" prints the braces literally.
  3. Adding a number to text directly: "Age: " + 19 crashes — convert with str() or use an f-string.
  4. Mis-spelling a variable name later than where you defined it — Python is case-sensitive.
  5. Forgetting the colon : after if, for, or a def line.
